Download & Installation Guide

Learn how to download and run VarStream on your device. VarStream ships as a standalone executable — no Python or extra runtime required for end users.

Windows: Download & Run VarStream

System Requirements: Windows 10 or later (64-bit), 4GB RAM, ~50MB disk space.
No Python needed: VarStream is a standalone .exe compiled with Nuitka.
1

Download VarStream EXE

Download VarStream-v1.0.0-windows-x64.exe from the VarStream download page.

2

Verify SHA256 Checksum (Optional)

Open PowerShell and run the following command to verify your download:

Get-FileHash -Algorithm SHA256 -Path "C:\Downloads\VarStream-v1.0.0-windows-x64.exe"

Compare the output with the checksum listed on the download page.

3

Run VarStream

VarStream for Windows is a command-line tool. You can:

  • Double-click VarStream-v1.0.0-windows-x64.exe to see the help message, or
  • Open Command Prompt / PowerShell and run:
cd C:\path\to\VarStream
.\VarStream-v1.0.0-windows-x64.exe myscript.vat

This will execute the myscript.vat VarStream file.

4

(Optional) Add VarStream to PATH

  1. Move VarStream-v1.0.0-windows-x64.exe to a permanent folder, e.g. C:\VarStream
  2. Add C:\VarStream to your system PATH
  3. Now you can run VarStream-v1.0.0-windows-x64.exe from any folder
5

Test with a Sample Script

Create a file test.vat:

Let name be "VarStream"
show name

Run:

VarStream-v1.0.0-windows-x64.exe test.vat

You should see VarStream printed in the console.

Troubleshooting (Windows)

For new tools, Windows SmartScreen may show a warning. Click "More info" and then "Run anyway" to allow VarStream to run.

This is normal for unsigned executables. Click "More info" and then "Run anyway". Make sure you downloaded VarStream from the official download page.

If you see permission errors, try:

  • Right-click the EXE and select "Run as administrator"
  • Move VarStream to a folder where you have full access (e.g. C:\Users\YourName\Tools)

Uninstallation (Windows)

VarStream is portable. To "uninstall":

  1. Close all terminals using VarStream
  2. Delete VarStream-v1.0.0-windows-x64.exe
  3. Remove any PATH entries you created manually (if applicable)

macOS: Download & Run VarStream

System Requirements: macOS 10.14 (Mojave) or later, 4GB RAM, ~50MB disk space.
VarStream is distributed as a standalone binary or app bundle (no system-wide Python required for users).
1

Download VarStream for macOS

Download VarStream-v1.0.0-macos.dmg (or .zip / .tar.gz variant) from the download page.

2

Verify SHA256 Checksum (Optional)

Open Terminal and run:

shasum -a 256 ~/Downloads/VarStream-v1.0.0-macos.dmg

Compare with the checksum shown on the download page.

3

Install / Extract VarStream

If using DMG:

  • Double-click the .dmg file to mount it
  • Drag the VarStream app or binary into your Applications folder

If using a ZIP/TAR:

  • Extract the archive
  • Move the VarStream binary to a folder like /Applications/VarStream or ~/Tools
4

Run VarStream from Terminal

Open Terminal and run:

cd /path/to/VarStream
./VarStream-macos myscript.vat

Replace VarStream-macos with the actual binary or app runner name used in your build.

5

First Launch Permissions

On the first run, Gatekeeper may ask for confirmation because VarStream is not from the App Store. Click "Open" to continue.

Troubleshooting (macOS)

Right-click the VarStream app or binary and choose "Open", then click "Open" again in the dialog.

You can also go to System Settings → Privacy & Security and click "Open Anyway" near the VarStream message.

This is usually a corrupted download or extended attributes issue. Re-download VarStream, then run:

xattr -cr "/Applications/VarStream.app"

Make it executable:

chmod +x /path/to/VarStream

Uninstallation (macOS)

To remove VarStream:

  1. Quit all Terminal sessions using VarStream
  2. Delete the VarStream app/binary from /Applications (or your custom folder)
  3. Optionally remove any VarStream-related config or scripts from your home directory

Linux: Download & Run VarStream

System Requirements: Linux distribution with glibc 2.17+, 4GB RAM, ~50MB disk space.
Recommended: 64-bit modern distro (Ubuntu, Debian, Fedora, etc.). VarStream is provided as AppImage / .deb / .rpm / tarball.

AppImage (Portable VarStream)

1

Download AppImage

Download VarStream-v1.0.0-x86_64.AppImage from the download page.

2

Make AppImage Executable

Open a terminal and run:

chmod +x ~/Downloads/VarStream-v1.0.0-x86_64.AppImage
3

Run VarStream

Execute the AppImage and pass a .vat script:

cd ~/Downloads
./VarStream-v1.0.0-x86_64.AppImage myscript.vat
4

(Optional) Install System-Wide

Move the AppImage and create a symlink:

sudo mv ~/Downloads/VarStream-v1.0.0-x86_64.AppImage /opt/varstream.AppImage
sudo ln -s /opt/varstream.AppImage /usr/local/bin/varstream

Now you can run:

varstream myscript.vat

.deb Package (Debian/Ubuntu)

1

Download .deb Package

Download varstream_1.0.0_amd64.deb from the download page.

2

Install VarStream

cd ~/Downloads
sudo dpkg -i varstream_1.0.0_amd64.deb
3

Fix Dependencies (If Needed)

sudo apt-get install -f

.rpm Package (Fedora/RHEL)

1

Download .rpm Package

Download varstream-1.0.0.x86_64.rpm from the download page.

2

Install VarStream

cd ~/Downloads
sudo rpm -i varstream-1.0.0.x86_64.rpm

On Fedora you may use:

sudo dnf install varstream-1.0.0.x86_64.rpm

Tarball (Generic Linux)

1

Download Tarball

Download VarStream-v1.0.0-linux-x64.tar.gz from the download page.

2

Extract the Archive

cd ~/Downloads
tar -xzf VarStream-v1.0.0-linux-x64.tar.gz
3

Move VarStream to /opt

sudo mv VarStream-v1.0.0-linux-x64 /opt/varstream
4

Create a Symlink

sudo ln -s /opt/varstream/varstream /usr/local/bin/varstream

Now run:

varstream myscript.vat

Common Linux Issues

Make sure VarStream is executable:

chmod +x varstream

On Debian/Ubuntu, install common GUI/runtime libraries if needed:

sudo apt-get update
sudo apt-get install libgtk-3-0 libnotify4 libnss3 libxss1 libxtst6 \
libatspi2.0-0 libdrm2 libxcomposite1 libxdamage1 libxrandr2 libgbm1 \
libxkbcommon0 libasound2

VarStream requires a relatively recent glibc. Try a newer distribution or use the AppImage build if available.

Common Download & Run Issues

Quick fixes for frequently encountered VarStream setup problems

Insufficient Disk Space

VarStream itself is small, but you need some free space for logs, scripts, and updates.

Solution: Keep at least 500MB free on the drive where you store VarStream and your .vat scripts.

Antivirus Flags the EXE

Some antivirus tools flag new unsigned executables, even when they are safe.

Solution: Verify the checksum, then temporarily allow or whitelist VarStream in your antivirus settings.

Corrupted Download

Unstable internet can cause incomplete or corrupted downloads, leading to errors when running VarStream.

Solution: Re-download VarStream over a stable connection and compare the SHA256 checksum with the value from the download page.

Command Not Found

Trying to run varstream from anywhere but the shell can't find it.

Solution: Either run VarStream from its folder (e.g., ./VarStream-v1.0.0-windows-x64.exe) or add it to your system PATH as described in the platform sections above.
