Quick Start Guide

From zero to running your first VarStream script

1

Download VarStream

Go to the Download page and download the VarStream executable for your platform (Windows, macOS, or Linux). No Python is required for end users.

2

Open a Terminal / Command Prompt

VarStream is a command-line tool:

  • Windows: Command Prompt or PowerShell
  • macOS: Terminal
  • Linux: Any terminal emulator

Navigate to the folder where you placed VarStream (or VarStream.exe / AppImage / binary).

3

Create Your First .vat File

Create a file named hello.vat with the following content:

Let name be "VarStream"
show name

Save it in the same folder as your VarStream executable, or remember the path.

4

Run the Script

In your terminal, run:

./VarStream hello.vat

(On Windows, it might be VarStream.exe hello.vat.) You should see:

VarStream
5

Explore Basic Syntax

Try adding more lines to your script:

x = 10
If x > 5 then show "Big"
If 2 > 5 then show "Wrong" else show "Good job"

Tip: Keep one instruction per line. VarStream is designed for simple, readable, English-like commands.

How VarStream Works

Understanding the basic flow: script → VarStream engine → output

A. Command Line

Your terminal or command prompt where you run VarStream and see the output.

B. Current Directory

The folder that contains your .vat scripts and, optionally, the VarStream executable.

C. Script File

Plain-text file with extension .vat (e.g. hello.vat) containing VarStream instructions.

D. VarStream Engine

The VarStream binary parses each line, normalizes natural language, and executes the corresponding action.

E. Output & Errors

Anything you show and any error messages are printed back to the terminal.

F. Script State

In-memory variables, lists, dictionaries, and function results that exist while the script is running.

Typical VarStream Workflow

1. Write a Script

  • Use any text editor (VS Code, Notepad, nano, etc.).
  • Save your file with .vat extension.
  • Keep syntax simple and English-like.

2. Run the Script

  • Open a terminal in the script folder.
  • Use VarStream yourscript.vat (or platform-specific executable name).
  • Check the output and error messages.

3. Iterate

  • Change your script, save, and run again.
  • Add variables, conditions, lists, dictionaries, and functions step by step.
  • Use errors as hints to fix syntax or logic.

Core Workflows

Common tasks you’ll do with VarStream

Writing and Running a Script

Create a simple VarStream file and execute it on Windows, macOS, or Linux.

1

Create the File

  • Create example.vat in your project folder.
  • Add basic instructions:
x = 9
y = 8
Check if x equals y
If 5 > 2 then show "Correct"
2

Run the File

  • Open a terminal in the same folder.
  • Run (example):
./VarStream example.vat

On Windows:

VarStream.exe example.vat
3

Interpret the Output

Check what is printed by show lines. If there is a syntax error, VarStream will stop and tell you which line is problematic.

Working with Input and Variables

Make scripts interactive using built-in input sentences.

1

Ask for User Input

Ask user for their name
Take input for age
show name
show age

This automatically stores values in name and age variables.

2

Use Math and Conditions

age = 18
If age >= 18 then show "Adult" else show "Minor"

Supported operators: + - * / > < >= <= == !=.

3

Combine with Lists & Dicts

Make a list called fruits containing apple, banana, orange
Add item mango to list fruits
create dict student : name "Keerthi", age 21
set_key student age 22
show fruits
show student

Organising Your VarStream Scripts

Keep your scripts tidy as you build more examples.

1

Use a Scripts Folder

  • Create a varstream-scripts directory.
  • Store each example as its own .vat file.
  • Group related scripts into subfolders (e.g. basics/, functions/, lists/).
2

Use Meaningful Names

  • input_demo.vat
  • conditions_basic.vat
  • functions_add.vat
3

Keep Comments in a Separate Doc

Since VarStream basic version doesn’t support comments yet, keep notes in a separate .txt or markdown file describing what each script does.

Language Features

Use the full power of basic VarStream syntax

Functions

Group reusable logic with simple function definitions.

  • Define function add(a,b) as set result to a+b; return result
  • Only one return per function (basic version)
  • No loops inside functions (in this prototype)
  • Call with: Call add with 5,3

Lists & Dictionaries

Store multiple values and structured data.

  • Ranges: Create a list of numbers 1 to 5
  • Custom list: Make a list called fruits containing apple, banana, orange
  • Dictionary: create dict student : name "Keerthi", age 21
  • Update key: set_key student age 22

Repeat Loops

Run a simple action multiple times.

  • repeat 5 times show "Hello"
  • repeat 3 times set x as x+1
  • Loops are single-line in the basic engine
  • Use them for simple counters and repeated messages

Good Practices

Keep VarStream scripts clean and understandable.

  • Use clear variable names: age, total, student_name
  • One instruction per line
  • Group related logic together (inputs → processing → output)
  • Start with simple scripts and grow slowly

CLI Tips

Helpful terminal habits when using VarStream

Running VarStream

Show help VarStream --help
Run a script VarStream script.vat
Run with debug (if enabled) VarStream --debug script.vat
REPL mode (if available) VarStream --repl

Common Terminal Keys

Interrupt running script Ctrl + C
Previous command Up Arrow
Next command Down Arrow
Clear screen clear (Linux/macOS) / cls (Windows)

File Operations (Shell)

List files ls (Linux/macOS) / dir (Windows)
Change directory cd foldername
Show current directory pwd (Linux/macOS)
Run script in current folder ./VarStream script.vat or VarStream.exe script.vat

Editing Scripts Faster

Open file in VS Code code script.vat
Open file in nano (terminal editor) nano script.vat
Duplicate script as backup cp script.vat script_backup.vat
View file quickly cat script.vat (Linux/macOS) / type script.vat (Windows)

Tip

Create a dedicated varstream folder with all your scripts, and run your terminal from there to keep everything organised.

Troubleshooting

Common VarStream issues and how to fix them

VarStream Won't Run

Check Platform

Make sure you downloaded the correct build for your OS (Windows/macOS/Linux) from the download page.

Executable Permissions (Linux/macOS)

Run:

chmod +x VarStream

Path Issues

Try running with a relative path from the folder where the file is stored:

./VarStream --help

"Command Not Found"

Run from the File’s Folder

Navigate to the folder containing VarStream:

cd /path/to/VarStream/folder

Use Relative Path

On Linux/macOS:

./VarStream script.vat

On Windows (Command Prompt):

VarStream.exe script.vat

Add to PATH (Optional)

If you want to run VarStream from anywhere, add its location to your system PATH (see OS-specific docs).

Syntax / Parsing Errors

Check the Line Number

The error usually shows which line VarStream couldn’t understand. Inspect that line in your .vat file.

Use Supported Patterns Only

Keep sentences close to supported forms, e.g.:

  • Set x as 10
  • If 5 > 2 then show "OK"
  • Make a list called fruits containing apple, banana

One Instruction Per Line

Avoid mixing multiple unrelated commands in a single line (except simple function definitions in the basic style you’ve defined).

Input or Show Not Working as Expected

Check Variable Names

Make sure the variable name you show matches exactly the one you used when taking input.

Ask user for their name
show name  # not 'Name'

Remember Input is String by Default

User input is read as text. If you compare with numbers, make sure you handle that in your design (e.g., use numeric defaults where needed).

Run in a Real Terminal

Some IDE consoles behave differently with input. If something feels off, test in a normal system terminal.
